# TASK: 모달 시스템 기능 고도화 (ESC, 뒤로가기, 백드롭 제어)

## WHAT
사용자 경험(UX) 향상을 위해 전역 모달 시스템에 다음 3가지 핵심 기능을 추가했습니다:
1. **ESC 키 대응**: 키보드 Escape 키를 눌러 활성화된 모달을 즉시 닫을 수 있습니다.
2. **뒤로가기 가로채기**: 브라우저의 뒤로가기 버튼 클릭 시 페이지를 이동하는 대신 열려있는 모달만 닫도록 구현했습니다.
3. **백드롭 클릭 제어**: `preventCloseOnClickBackdrop` 옵션을 통해 배경 클릭 시에도 모달이 닫히지 않도록 설정할 수 있는 기능을 추가했습니다.

## WHY
- **웹 접근성 및 편의성**: 키보드 조작을 통한 모달 제어는 현대적인 웹 서비스의 필수적인 접근성 요소입니다.
- **네이티브 앱 경험**: 모바일 브라우저나 뒤로가기 버튼에 익숙한 사용자가 의도치 않게 페이지를 벗어나는 것을 방지하고, 직관적인 네비게이션 경험을 제공하기 위함입니다.
- **데이터 보호**: 결제나 중요한 설정 단계에서 모달 외부 클릭으로 인해 작업 내용이 유실되는 것을 방지할 수 있는 제어 장치가 필요했습니다.

## HOW & WHY
- **History API 활용**: 모달 마운트 시 `window.history.pushState`를 사용하여 가상 히스토리를 쌓고, `popstate` 이벤트를 리스닝하여 뒤로가기 동작을 감지합니다. 
- **역방향 동기화**: 모달 내부 버튼 등으로 직접 닫을 때도 히스토리 스택을 깨끗하게 유지하기 위해 `window.history.back()`을 호출하여 가상 상태를 정리합니다.
- **Hook-based Lifecycle**: `ModalRenderer`의 `useEffect`를 통해 이벤트 리스너와 히스토리 관리를 캡슐화하여, 여러 개의 모달이 겹쳐있을 때도 스택 구조로 정확하게 작동하도록 설계했습니다.

## RESULT
- `ui.modal.custom(Comp, opts, { preventCloseOnClickBackdrop: true })` 형태로 고급 옵션 사용 가능.
- 브라우저 뒤로가기 시 모달이 순차적으로 닫힘.
- `npm run build` 결과 모든 타입 체크 통과.
