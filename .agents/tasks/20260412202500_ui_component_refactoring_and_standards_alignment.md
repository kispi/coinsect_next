# Task: UI 컴포넌트 리팩토링 및 표준 준수

## 무엇을 했는가?
- 하드코딩된 토글 로직을 공통 컴포넌트 `AppToggler.tsx`로 추출
- `Dropdown` 컴포넌트를 `WrapperDropdown`으로 명칭 변경 및 일관성 확보
- 모든 프로젝트 컴포넌트의 루트 클래스명을 `dash-case` 명명 규칙으로 통일
- 이전 작업 내역에 대한 정식 태스크 기록 생성

## 왜 그렇게 했는가?
1. **컴포넌트 재사용성**: `ModalPositionRequestEdit.tsx`에서 개별적으로 구현되어 있던 토글 스위치를 `AppToggler`로 공통화하여 코드 중복을 줄이고 유지보수성을 높였습니다. 이는 Vue 원본 코드의 `AppToggler.vue` 패턴을 따른 것입니다.
2. **명확한 명명 규칙**: 기존의 `Dropdown`은 단순히 레이아웃을 감싸는 오버레이 역할을 하므로, 그 역할을 더 명확히 드러내는 `WrapperDropdown`으로 변경했습니다. 이는 Vue의 `WrapperDropdownOverlay`와의 연계성을 고려한 결정입니다.
3. **프로젝트 표준 준수 (`general.md`)**:
   - `dash-case` 루트 클래스 명명 규칙(ex: `className="app-header ..."`)을 철저히 적용하여 CSS 스타일링의 일관성을 확보했습니다.
   - 작업 내역을 `.agents/tasks/` 폴더에 정해진 형식에 맞춰 기록함으로써 프로젝트 히스토리 관리 체계를 정립했습니다.

## 기술적 결정 사항
- **AppToggler 구현**: Tailwind CSS의 `transition`과 `transform`을 활용하여 라이브러리 없이 부드러운 토글 애니메이션을 구현했습니다. `small` 옵션을 지원하여 다양한 UI 상황에 대응 가능하게 했습니다.
- **일괄 리팩토링**: `AppHeader.tsx` 등 다수의 컴포넌트에서 혼용되던 명칭과 클래스 구조를 전수 조사하여 프로젝트 가이드라인에 맞게 수정했습니다.

## 결과
- 컴포넌트 구조가 더 체계적으로 변했으며, 향후 기능 추가 시 일관된 패턴을 따를 수 있는 기반이 마련되었습니다.
- 프로젝트 문서화 규칙을 엄격히 준수하게 되어 작업의 투명성과 연속성이 향후에 보장됩니다.
