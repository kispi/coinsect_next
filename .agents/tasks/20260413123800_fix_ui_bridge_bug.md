# TASK: UI Utility (Toast, Modal) 작동 불능 버그 수정

## WHAT
`ui.toast`, `ui.modal`, `ui.snackbar` 등 컴포넌트 외부에서 사용하는 전역 UI 유틸리티가 작동하지 않던 버그를 수정하고, 관련하여 발생한 `AppHeader.tsx`의 타입 오류를 해결했습니다.

## WHY
1. **브릿지 초기화 불일치**: 스토어를 Provider 패턴으로 전환하면서 `setUIStore`를 `useState` 이니셜라이저에서 호출하도록 수정했으나, 이 코드가 서버 사이드에서도 실행되면서 클라이언트 측 전역 변수가 올바르게 초기화되지 않는 문제가 발생했습니다.
2. **타입 안정성 확보**: 이전 작업에서 `SettingsPanel.tsx`의 미사용 props를 제거했으나, 이를 호출하는 `AppHeader.tsx`에서 여전히 해당 props를 넘겨주고 있어 전체 빌드가 깨지는 현상을 발견하여 수정이 필요했습니다.

## HOW & WHY
- **Client-Side Effect 초기화**: `setUIStore` 호출을 `StoreProvider`의 `useEffect` 내부로 이동했습니다. 이는 React의 하이드레이션이 끝난 뒤 클라이언트 사이드에서만 안전하게 스토어 인스턴스를 정적 브릿지에 연결하도록 보장합니다.
- **Interface 정규화**: `SettingsPanel`의 `onClose` props를 사용하는 대신, 부모 컴포넌트인 `WrapperDropdown`의 상태 관리 로직을 따르도록 호출부를 정리했습니다.

## RESULT
- 브라우저 환경에서 `ui.toast` 및 `ui.modal` 기능이 다시 정상적으로 작동합니다.
- `npm run build` 결과 모든 타입 체크와 정적 페이지 생성이 성공적으로 완료되었습니다.
